# UX Test Patterns

模拟用户测试时容易被通用 checklist 漏掉的具体失败形状库——每条 pattern 描述"失败长什么样"。遇到新形状可以新增 pattern。

---

## P1. 时间线排序语义

**Ask yourself**：这个列表的命名、视觉或控件是否在向用户暗示"按时间倒序"？

**形状**：列表的可见顺序与 UI 暗示的排序口径不一致——叫"timeline / 最新 / 动态 / 全部"却按分数或推荐排序；API 按时间返回但前端按分数重排没有提示；过滤/分页后顺序被悄悄换；日期分组内部时间逆序但 UI 没解释；跨页边界出现逆序或重复。

## P2. 分类 / 标签与内容语义匹配

**Ask yourself**：分类名或条目标签让普通用户预期"这里该出现什么"——实际看到的内容会让他困惑吗？

**形状**：分类或标签的名字与内容的实际语义不符——分类下混入与该分类名语义擦边或无关的内容（"教程"被算进"模型"、靠多个弱相关 tag 把内容拉错地方）；单条 item 挂的标签精确但内容只擦边；标签命名颗粒度（领域级）与内容颗粒度（产品级）错位；同一概念在不同位置叫不同名字；同一分类在前端过滤、后端过滤、深链接下口径不一致；URL 是某分类但 active tab / 搜索提交后的分类参数丢失。

## P3. 深链接 / 点击路径 / API 结果一致性

**Ask yourself**：同一逻辑状态可以由"点击进入"和"直接打开深链接"两条路径到达吗？如果可以，两条路径下结果可能不同吗？

**形状**：同一筛选 / 分页 / 查询状态在不同到达路径下结果不一致——直链初始 HTML 与 JS 接管后短暂错位，造成闪烁或无 JS 失败；刷新或后退/前进后控件不同步；API 与页面口径不齐；无效参数处于半 active 半默认的中间状态而非可预测降级。

## P4. 视觉隐藏的元素仍可被键盘/读屏摸到

**Ask yourself**：页面里有"视觉看不见但 DOM 仍存在"的容器吗（移动 sidebar、drawer、modal、overflow 菜单、toast）？关闭状态下用键盘 Tab 或读屏器会摸到里面的控件吗？

**形状**：视觉上关闭的容器里的控件仍出现在 tab order 或 accessibility tree 中——sidebar 用 `translateX(-100%)` 隐藏但链接仍在 tab order 中；modal 背后的页面仍可键盘聚焦；菜单视觉关闭但 screen reader 仍读到菜单项；打开 drawer 后焦点没有进入容器，关闭后焦点没有回到触发按钮。

## P5. 自然点击目标和返回路径

**Ask yourself**：在"用户点列表/卡片 → 消费 → 返回"这条行程上，下一步意图能否顺利续上？（覆盖：自然想点的元素是否可点、返回时 filter/滚动/搜索/分页是否保留）

**形状**：卡片或列表中"显眼且语义最强的元素"不可点，逼用户去找小图标（只有小图标可点、标题不可点；图片可点但标题不可点）；或点击后返回时上下文丢失（分类、滚动位置、搜索词被清空），用户被迫从头重建筛选状态；链接打开方式与心智不符（站内详情却跳新窗口、外链却在原窗口）。

## P6. 加载中、空结果、错误状态可区分

**Ask yourself**：异步加载、查询无结果、API 失败——这三种本质不同的"目前看不到内容"，用户能从屏幕上区分吗？

**形状**：三种状态被同一个空白或同一句"暂无内容"吸收——首次访问 API 等待期间列表为空让用户以为没内容；空搜索结果和网络错误共用"暂无内容"；API 失败永远 skeleton 或静默空白；分页最后一页或无效参数没有可回退的路径。

## P7. 窄屏和缩放下保持同一阅读心智

**Ask yourself**：在窄屏、浏览器缩放或大系统字号下，用户的扫读顺序、控件可达性、首屏看到什么——是否仍像同一个产品？

**形状**：重排后页面失去原来的视觉优先级——标题被挤、控件被隐藏在横向滚动或被压扁、图片占满首屏导致看不到正文；缩放档位下标题断行或控件相互遮挡；移动断点切到了但参照产品已切而本产品没切（或反之）。即使没有横向滚动，扫读路径也可能已经崩坏。

## P8. 内容新鲜度与承诺节奏吻合

**Ask yourself**：页面或品牌承诺了某种更新节奏（"最新" / "每日" / "实时" / "Today"）吗？可见发布时间与那个承诺吻合吗？

**形状**：可见发布时间显著落后于产品承诺的更新频率——标题或分类暗示"现在"但实际内容来自更早；列表页时间刷新但详情页时间字段卡在旧值；时间戳精度过粗让用户无法区分"刚刚"和"几小时前"。

## P9. 去重与事件聚合

**Ask yourself**：同一事件、同一新闻、同一研究——产品是否承担"合并 / 关联"的职责？还是任由多条独立条目并排出现？

**形状**：同一事件被拆为多条互不引用的独立条目，列表浏览像在重复消费同一内容；不同源对同一事件各占一条无聚合视图；同义标题/链接产生独立 entry；短窗口内同一作者/来源刷屏没有节流。

## P10. 来源可信度与摘要保真

**Ask yourself**：对高影响内容（融资、人事、研究结论、安全事件），点进去能验证摘要与原文一致吗？

**形状**：摘要、标题、标签在原文或来源面前夸大、断章或张冠李戴——摘要夸大原文结论；标题与正文情绪/事实不符；来源页面已删除或与摘要无关但产品没有降级处理；引用来源不可信（个人推文被当成官方公告）。
